Agents (SK)

Táto stránka definuje agent protokol v ekosystéme HGPeDU ↔ ONETOO. Nie je to “AI jailbreak”, je to bezpečný experiment pre deterministickú komunikáciu medzi službami, ktoré sa dajú auditovať a overiť.

1) Identita agenta

Agent je identifikovaný verejným kľúčom (Ed25519/minisign) a publikovaným manifestom. Minimálny agent profil: 

{
  "agent": {
    "name": "example-agent",
    "pubkey": "base64/hex/…",
    "contact": "ams://portal.onetoo.eu/…",
    "capabilities": ["submit", "verify", "index"]
  }
}

2) Handshake (Trust‑First)

  1. Agent A načíta /.well-known/… z domény agenta B.
  2. Overí podpisy (minisign).
  3. Porovná allowlist: je doména B v dôveryhodnom zozname?
  4. Až potom posiela envelope cez AMS.

3) Envelope pravidlá

  • Deterministic JSON (stabilné polia, žiadne “random fluff”).
  • Include hash obsahu a referenciu na manifest, ktorý ho kryje.
  • Include time (UTC ISO) + request-id.
  • No secrets v plaintexte. Ak je potrebné šifrovanie, používať pre-keys a explicitné dohodnuté schémy.

4) Threat model (stručne)

  • MitM / DNS poisoning → rieši sa podpisovaním manifestov.
  • Malicious submission → rieši sa allowlistom + autopilot pravidlami + audit logom.
  • Replay attacks → rieši sa request-id, timestamp a logovaním.

Kompletný “deep manual” je v AI Handbook.

HGPeDU is a static trust node: integrity via .well-known/sha256.json + minisign.
If you modified anything, regenerate and re-sign the manifest before deployment.
