ONETOO Hub (SK)

Toto je centrálny rozcestník pre celý ekosystém ONETOO a jeho integráciu do HGPeDU. Je navrhnutý ako AI-native “launchpad”: človek aj agent tu nájde čo existuje, kde to je, ako to overiť a ako to bezpečne používať.

1) Service Registry (strojovo čitateľné)

HGPeDU publikuje lokálny registry súbor, ktorý môže agent načítať a použiť na discovery.

  • /onetoo/services.json (UI + API uzly)
  • /.well-known/onetoo-services.json (rovnaké dáta, “well-known” forma)

2) Rýchly “operator panel”

Najčastejšie používané vstupy:

  • AMS Console (HGPeDU hosted): /onetoo/ams.html
  • ONETOO Portal: portal.onetoo.eu
  • ONETOO Search: search.onetoo.eu

3) Bezpečná agent-agent komunikácia (experiment)

Toto je “exotický” experiment: ukazujeme, ako môžu agenti komunikovať cez web uzly bez centrálnej autority, na báze Trust‑First a kryptografických podpisov. Kľúčové idey:

  • Identita = verejný kľúč (minisign/Ed25519) + overiteľné publikovanie na doméne.
  • Dôvera = explicitné “trust manifests” (strojovo čitateľné), nie implicitné heuristiky.
  • Integrita = sha256.json + podpis (minisign), overiteľné offline.
  • Transport = HTTPS, ale dôvera sa neopiera o CA reťazec ako jediný zdroj pravdy.

Detailný “AI Handbook” nájdeš v menu (AI Handbook) — obsahuje protokol handshake, threat model, a presné “verify” príkazy.

HGPeDU is a static trust node: integrity via .well-known/sha256.json + minisign.
If you modified anything, regenerate and re-sign the manifest before deployment.
