Full integration (Portal Autopilot + ONETOO AI Search Autopilot)

This site is static, but it behaves like a living system: contributions arrive via GitHub Issues, are validated, published to the portal feed, and then automatically submitted to ONETOO AI Search (global index).

Autonomous pipeline

Issue → validate JSON → publish /data/portal/index.json → submit to search.onetoo.eu → track status → rebuild

Roadmap

  • Now: Issue-driven publishing to /data/portal/index.json (no human step).
  • Next: Automatic submit to ONETOO contrib API and local submission ledger /portal/submissions.json.
  • Next: Display ONETOO acceptance status in the portal table (accepted / pending / rejected).
  • Optional: Signed payloads (Minisign) using GitHub Actions secrets for fully auditable provenance.

Where the data lives

  • Portal feed: /data/portal/index.json
  • Submission store: /portal/submissions.json

How users contribute

  1. Open a GitHub Issue titled [PORTAL] ...
  2. Paste one JSON object (template is provided)
  3. Autopilot does the rest

Important

ONETOO submission requires an API token. Store it as a GitHub Actions secret (e.g. ONETOO_TOKEN). This keeps the pipeline autonomous while protecting credentials.
