HGPeDU ↔ ONETOO: Holy‑Grail integration

Tento dokument je „svätý grál“ pre to, ako sa má statická TFWS v2 stránka (hgpedu.eu) správať ako plnohodnotný uzol v ekosystéme ONETOO: publikovateľná, auditovateľná, indexovateľná, decentralizovaná.

Pipeline (bez backendu)

Issue/PR → validate JSON → publish portal feed → publish TFWS trust surface → notify ONETOO maintainers → indexed (accepted-set) → searchable

Cieľ: pridať „záložku“ do ONETOO vyhľadávania tak, aby bola dôveryhodná (TFWS), opakovateľná a podpisovo auditovateľná.

⚠️ ONETOO štandard 2026: verejný backend je read‑only

search.onetoo.eu dnes publikuje GET‑only endpointy (napr. /search/v1) a nezverejňuje verejný POST /contrib/…/submit endpoint. Príspevky sa teda kurátorsky pridávajú na strane ONETOO (accepted‑set), kým HGPEdu drží transparentný lokálny audit (portal feed + integrity).

Test: https://search.onetoo.eu/search/v1?q=hgpedu&limit=10

1) Priprav TFWS v2 trust‑surface (na webe)

  1. Publikuj trust hub: /ai-trust-hub.json + podpis .minisig.
  2. Publikuj verejný kľúč: /minisign.pub (a prípadne aj ed25519 pubkey, ak používaš viac systémov).
  3. Publikuj integrity: /SHA256SUMS.txt (+ podpis) pre archívne balíky a dôležité súbory.
  4. Publikuj sitemap: /sitemap.xml a umožni indexovanie cez /robots.txt.

2) Priprav „záložku“ (contribution record)

Odporúčaný minimálny záznam (TFWS‑štýl JSON)

Neobsahuj nebezpečné návody / HV parametre. Záznam má byť o publikácii, metodike, dátach a verifikácii.

{
  "type": "onetoo.contribution",
  "version": "2.0",
  "date": "YYYY-MM-DDTHH:MM:SSZ",
  "title": "HGP-PHF · Research page (publication-safe)",
  "url": "https://hgpedu.eu/pages/research.html",
  "summary": "Archival sci-fi technical narrative + verbatim sources, TFWS v2 trust surface, safe publication framing.",
  "categories": ["research", "tfws", "hgp-phf"],
  "license": "CC BY 4.0",
  "safety": {"no_build_instructions": true, "no_hv_schematics": true}
}

3) Odoslanie do HGPeDU Portalu (GitHub)

  1. Na Portal 2.0 vygeneruj JSON.
  2. Odošli ako GitHub Issue (alebo PR do /data/portal/submissions/).
  3. Po merge sa záznam objaví vo feede (/data/portal/index.json).

4) Odoslanie do ONETOO (contrib API)

Autopilot režim (odporúčané)

  • GitHub Actions workflow načíta nové portal záznamy.
  • Validuje JSON (schema + pravidlá bezpečnosti).
  • Odošle do search.onetoo.eu contrib endpointu.
  • Zapíše lokálny ledger (submission log) a zobrazí status v tabuľke portalu.

Token sa drží ako GitHub Secret (napr. ONETOO_TOKEN). Nikdy sa nepíše do repo.

5) Čo ONETOO indexuje

  • URL + meta: title/description, jazyk, kategórie.
  • Trust surface: ai‑trust‑hub + minisign pubkey + integrity.
  • Audit: dátum, verzia, podpisy, prepojenia na zdroje a balíky.

6) Kontroly kvality („svätý grál“ checklist)

Indexing

  • Sitemap obsahuje research/portal/downloads.
  • robots.txt povoľuje crawlovanie.
  • Každá stránka má správny <title> a description.

TFWS v2

  • ai-trust-hub.json existuje a je podpísaný.
  • minisign.pub dostupný.
  • SHA256SUMS publikované (a podpis).

Safety

  • Žiadne build‑kroky / HV parametre.
  • Publikuj metodiku, dáta, negatívne výsledky.
  • Jasné disclaimer texty + Safety stránka.
