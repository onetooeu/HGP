Specs (SK)

Špecifikácie, “living docs” a odkazy na nástroje. Toto je miesto pre technických ľudí aj agentov, ktorí potrebujú jednoznačné pravidlá.

1) TFWS v2

  • TFWS v2 dokumentácia: /pages/tfws.html
  • Adoption kit a tools sú priamo v GitHub repozitároch (pozri ONETOO Hub).

2) HGPeDU manifesty

  • /.well-known/ai-hgpedu.json
  • /.well-known/ai-trust-hub.json
  • /.well-known/ai-governance.json
  • /.well-known/sha256.json

3) ONETOO discovery

  • Local registry: /onetoo/services.json
  • Well-known registry: /.well-known/onetoo-services.json

4) Recommended API shapes

  • NDJSON pre audit streamy (line-delimited JSON).
  • JSON pre snapshoty / indexy / manifests.
  • Deterministic sorting v indexoch, aby sa dali porovnávať hashom.

5) “Operator commands” (copy/paste)

# 1) manifest verify
minisign -V -p .well-known/minisign.pub -m .well-known/sha256.json -x .well-known/sha256.json.minisig

# 2) regenerate (after changes)
python tools/gen_sha256_json.py
bash tools/sign_minisign.sh

# 3) quick linkcheck (local)
python tools/linkcheck_local.py
HGPeDU is a static trust node: integrity via .well-known/sha256.json + minisign.
If you modified anything, regenerate and re-sign the manifest before deployment.
