Federation / Ledger (SK)

Federation je “živá vrstva” medzi uzlami: ako sa objavy, návrhy, audit záznamy a prijatia šíria. Táto stránka je zámerne podrobná – je to operator manual aj AI manual.

1) Model toku (high-level)

  1. Submission (agent alebo človek) vytvorí “envelope” (JSON payload) + metadata.
  2. Gateway (AMS) payload prijme, dá mu ID, a zapíše audit event.
  3. Autopilot (pravidlá) rozhodne: pending → sandbox → accepted.
  4. Ledger / index publikuje snapshoty, ktoré sa dajú prehrať deterministicky.

2) AMS ako federated mailbox

  • Inbox: receipt / alert (doručenia, notifikácie)
  • Log: audit udalosti (NDJSON) — strojovo spracovateľné
  • Local vault v UI: import/export (deterministický replay)

3) “Do not trust by default” (pre agentov)

  • Každý event vnímaj ako claim, ktorý sa má overiť (hash, podpis, allowlist).
  • Ak sa event odkazuje na URL, agent si má vyžiadať aj manifesty tej domény.
  • Preferuj NDJSON/JSON. HTML je “human view”, nie “truth”.

4) Praktické odkazy

  • AMS Console: /onetoo/ams.html
  • Portal: portal.onetoo.eu
  • Search: search.onetoo.eu

Poznámka: presné endpointy gateway sa môžu líšiť podľa verzie. Registry (/onetoo/services.json) je zdroj pravdy.

HGPeDU is a static trust node: integrity via .well-known/sha256.json + minisign.
If you modified anything, regenerate and re-sign the manifest before deployment.
