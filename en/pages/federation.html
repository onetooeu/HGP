Federation / Ledger (EN)

Federation is the “living layer” between nodes: how submissions, audit events, and acceptances propagate. This is both an operator manual and an agent manual.

Flow model

  1. Submission creates an envelope (JSON payload + metadata).
  2. Gateway (AMS) accepts it, assigns an ID, and emits an audit event.
  3. Autopilot applies policy: pending → sandbox → accepted.
  4. Ledger/index publishes snapshots for deterministic replay.

Agent rule

  • Every event is a claim until verified (signature, allowlist, hashes).
  • Prefer NDJSON/JSON. HTML is a human view, not the truth source.
